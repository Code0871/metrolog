## Описание директорий

### `config/`
YAML-файлы для настройки проекта. Содержат параметры, которые меняются между запусками:
- Пути к данным
- Гиперпараметры моделей
- Параметры preprocessing

### `data/`
**Не хранится в Git.** Содержит все данные для обучения и валидации:
- `external/` — справочные данные (реестры, ГОСТы)
- `features/` — признаки, сконструированные из сырых данных
- `processed/` — очищенные и нормализованные данные, готовые к обучению

### `doc/`
Техническая документация проекта. Здесь описывается:
- Установка и настройка окружения
- Структура проекта
- API (в будущем)

### `src/models/`
Исходный код моделей. Все модели наследуются от `base.py`:

| Файл | Назначение |
|------|------------|
| `base.py` | Абстрактный базовый класс, общий интерфейс |
| `borderline_type_classifier_model.py` | Multi-task модель: классификация по типу И погрешности |
| `lifecycle_model.py` | Классификация СИ по сроку службы |
| `purchase_model.py` | Прогнозная модель для планирования закупок |

### `training/`
**Не хранится в Git.** Все артефакты, генерируемые в процессе обучения:

| Папка | Назначение |
|-------|------------|
| `checkpoints/` | Сохранённые веса моделей (.pt, .pkl) |
| `logs/` | Логи обучения (TensorBoard, WandB, текстовые логи) |
| `metrics/` | JSON-файлы с метриками на валидации |
| `artifacts/` | Скейлеры, энкодеры, копии конфигов |
| `model_versions/` | Экспериментальные версии (для A/B тестирования) |
| `plots/` | Графики обучения, матрицы ошибок, визуализации |

## Примечания

- Папки `data/` и `training/` добавлены в `.gitignore` и не должны попадать в репозиторий
- После `pip install .` в корне могут появиться `build/` и `*.egg-info/` (уже добавлены в `.gitignore`)
- При разработке новые модели добавляются в `src/models/` и наследуются от `base.py`
