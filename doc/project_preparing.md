## Подготовка окружения проекта перед работой
Для работы с проектом нам потребуется:

#### Скопировать репозиторий:
Сделать это можно командой:
```bash
git clone https://github.com/Code0871/metrolog.git 
```
>**Примечание:**Универсальна для любой ОС с установленным git

#### Зайти в папку проекта
Используем команду:
```bash
cd metrolog
```
Или, если вы клонировали в другую директорию:

```bash
cd <путь к скопированному репозиторию>
```

#### Установка зависимостей:

Если вы используете Linux или другие UNIX-подобные ОС, можете столкнуться с проблемой запрета установки пакетов Python в системную версию языка, так как это может сломать зависимости установленных программ
Для решения этой проблемы нужно создать виртуальное окружение с помощью следующей команды:

```bash
python -m venv .venv
```
Далее нужно активировать виртуально окружение, делается это следующей командой:
```bash
source .venv/bin/activate
```
После чего, в терминале вы увидете `(.venv) [<стандартное имя пользователя и т.д.>]$`

Теперь, мы можем безопасно установить зависимости следующей командой:
```bash
pip install .
```
>**Примечание:** Возможно у вас возникнет вопрос: "а почему не указываем зависимости в виде: `pip install catboost`?" потому что в Python появился стандарт `PEP 668` позволяющий нам описывать все необходимые зависимости и их версии в файле `pyproject.toml`, что на самом деле очень удобно
