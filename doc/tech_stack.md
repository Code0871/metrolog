# Технологический стек проекта Metrolog

## Обзор

Metrolog — система анализа и прогнозирования парка средств измерений (СИ), построенная на современном Python-стеке с акцентом на обработку данных, машинное обучение и надёжную серверную инфраструктуру.

## Машинное обучение

| Библиотека | Версия | Назначение |
|------------|--------|------------|
| **catboost** | 1.2.0+ | Градиентный бустинг для классификации и регрессии |
| **prophet** | 1.1.0+ | Прогнозирование временных рядов (закупки, списания) |

### Примечание по моделям:
- **CatBoost** — основная модель для классификации СИ по типу, погрешности и сроку службы. Выбран за:
  - Отсутствие необходимости в предварительном кодировании категориальных признаков
  - Высокую производительность на табличных данных
  - Встроенные инструменты интерпретации (SHAP, feature importance)

- **Prophet** — модель для прогнозирования закупок. Оптимальна для:
  - Временных рядов с сезонностью
  - Автоматического учёта праздников и аномалий

## Обоснование выбора ключевых технологий

| Технология | Обоснование выбора |
|------------|-------------------|
| **CatBoost** | Лучшая работа с категориальными признаками "из коробки", высокая производительность на табличных данных, интерпретируемость |
| **Prophet** | Специализация на временных рядах с сезонностью, устойчивость к выбросам, простота настройки |
| **FastAPI** | Высокая производительность, автодокументация (OpenAPI), асинхронность, типизация через Pydantic |
| **SQLAlchemy** | Гибкость, поддержка миграций (Alembic в будущем), независимость от конкретной СУБД |
| **Loguru** | Удобный API, цветной вывод, простое логирование в файлы |
