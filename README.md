## О проекте
**Metrolog** это _Multi-Task Learning (MTL)_ model, предназначенная преимущественно для **аналитической** работы с парком средств измерений (СИ)

---

## Цель проекта Metrolog:
Основной целью данного проекта является разработка multi-task модели для анализа и работы с парком средств измерений, что позволило бы сократить нагрузку на метрологические службы предприятий

### Для достижения поставленной цели требуется:
 - [ ] Разработать BackBone
 - [ ] Разработать Heads модели
 - [ ] Собрать данные для обучения моделей
 - [ ] Очистить собранные данные
 - [ ] Обучить модели
 - [ ] Валидировать модели (проверить метрики на тестовой выборке)
 - [ ] Сохранить артефакты (веса моделей, конфиги, скейлеры)
 - [ ] Разработать API для интеграции
 - [ ] Настроить базу данных (хранение истории прогнозов и метаданных СИ)
 - [ ] Реализовать логирование (отслеживание ошибок и событий системы)
 - [ ] Написать тесты (юнит-тесты для моделей и API)
 - [ ] Настроить мониторинг (отслеживание дрейфа данных и деградации моделей)

---

## Функционал 
Разрабатываемый функционал:
 - Классификация СИ по сроку службы с обозначением СИ, приближающихся к снятию с учёта
 - Классификация СИ по типу и погрешности
 - Планирование закупок (прогнозная модель)
 - Автоматическое определение типа СИ
